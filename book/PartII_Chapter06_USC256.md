PARTIE II — ARCHITECTURE USC‑96 / USC‑128 / USC‑256  
Chapitre 6 — USC‑256 : Le Corps Universel Étendu
================================================

> **Statut** : squelette + résumé, à détailler en lien avec `usc_256_table.md`.

6.1 Pour IA globales, systèmes complexes, scientifiques, militaires
-------------------------------------------------------------------

- Présenter USC‑256 comme couche complète pour :
  - IA globales,
  - systèmes multi‑domaines,
  - environnements critiques (défense, santé, énergie, espace).  
- Expliquer pourquoi l’extension disciplinaires est regroupée dans ce niveau (128 symboles supplémentaires).

6.2 Champs ajoutés
------------------

- Physique (`PHYS-*`)  
- Chimie (`CHEM-*`)  
- Biologie / Médecine (`BIO-*`, `MED-*`)  
- Cartographie spatiale (`SPACE-*`, `MAP-*`)  
- Topologie / graphes (`TOPO-*`, `GRAPH-*`)  
- Mathématiques avancées (`MATH2-*`)  
- Information / Calcul (`INFO-*`, `COMP-*`)  
- Meta / Discipline / Réservé (`META-*`, `DISC-*`, `RSV-*`)

6.3 L’USC‑256 comme “Unicode scientifique”
-----------------------------------------

- Argumenter que USC‑256 fournit un **noyau symbolique scientifique commun**,  
  comparable, pour les concepts, à ce que Unicode est pour les graphèmes.  
- Montrer comment les disciplines peuvent se greffer dessus sans réinventer un alphabet.

6.4 Interfaçage avec MML / DNF
------------------------------

- Décrire comment les symboles USC‑256 :
  - s’insèrent dans des structures MML (séquences, graphes, arbres),  
  - se sérialisent dans des formats compacts (DNF, binaire, etc.).  
- Fournir des exemples de trames complexes (mélange de symboles de plusieurs domaines).

6.5 Table complète (Annexe A)
-----------------------------

- Renvoyer à `usc_256_table.md` comme référence normative.  
- Indiquer que toute implémentation USC‑256 MUST respecter cette table.

6.6 Extension future USC‑512 (prospective)
------------------------------------------

- Esquisser des pistes pour une future extension :
  - domaines supplémentaires,
  - granularité accrue,
  - compatibilité stricte avec les IDs existants.  
- Souligner que **USC‑512 ne doit pas remettre en cause la stabilité des IDs 0–255**.
